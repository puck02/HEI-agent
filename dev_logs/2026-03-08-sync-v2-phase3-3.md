# Sync v2 Phase 3.3（删除闭环 + 冲突策略）

日期：2026-03-08

## 目标
- 打通 Sync v2 删除同步闭环（tombstone）
- 增加基础冲突策略（server-first：base 过旧则拒绝并要求 pull）
- 完成后端/Android 回归验证

## 代码变更

### 后端（HEI-agent）
1. 新增 tombstone 持久化模型：
   - `app/models/health_data.py`
   - 新增 `SyncTombstoneRecord`（`entity`、`record_id`、`payload_json`、`deleted_at`）

2. 扩展同步 schema：
   - `app/schemas/health.py`
   - `SyncTombstone` 新增 `payload`

3. 同步路由增强：
   - `app/api/v1/sync.py`
   - 新增 `_record_tombstone`、`_compute_server_cursor`
   - `push` 删除操作会写入 tombstone（health_entry / medication / medication_course）
   - `medication_course` 新增删除支持（按 `record_id`/`start_date`/`status` 匹配）
   - `push` 增加 stale base 冲突短路：`base_server_version < current_cursor` 时返回 conflict
   - `pull` 返回 tombstones（含 payload）并参与 `next_cursor` 计算
   - `status` capabilities 新增：`sync.tombstone.v1`、`sync.conflict.v1`

### Android（HEl）
1. 模型扩展：
   - `app/src/main/java/com/heldairy/core/network/agent/AgentModels.kt`
   - `SyncTombstone` 新增 `payload`

2. 回放逻辑增强：
   - `app/src/main/java/com/heldairy/core/data/DataSyncManager.kt`
   - `syncNow()` 中新增 `applyPulledTombstones()`
   - 新增删除回放：
     - `deleteHealthEntry`（优先按 `entry_date`）
     - `deleteMedication`（按 `android_id` / `name` / `record_id` 兜底）
     - `deleteMedicationCourse`（按 `record_id` 或 payload 组合条件）
