# 2026-03-08 Sync v2 实施记录（Phase 3）

## 本阶段目标
- 落地 Sync v2 协议骨架（push/pull/status 扩展）
- 保持与现有 v1 upload/status 兼容
- Android 客户端先接入 v2 能力，DataSyncManager 优先 push，失败回退 upload

## 后端改造（HEI-agent）

### 1) Schema 扩展
- 文件：`app/schemas/health.py`
- 新增：
  - `SyncChange`
  - `SyncPushRequest` / `SyncPushResponse` / `SyncPushResult`
  - `SyncEntityEnvelope` / `SyncTombstone` / `SyncPullResponse`
- 扩展：`SyncStatusResponse`
  - 新增 `server_cursor`、`capabilities`、`last_push_ack`

### 2) API 扩展
- 文件：`app/api/v1/sync.py`
- 新增端点：
  - `POST /api/v1/sync/push`
  - `GET /api/v1/sync/pull?since=&limit=`
- 扩展 `GET /api/v1/sync/status` 返回能力字段
- 保留原有 `POST /api/v1/sync/upload`，兼容旧客户端

## 客户端改造（HEl）

### 1) 网络模型扩展
- 文件：`core/network/agent/AgentModels.kt`
- 新增：
  - `SyncPushRequest/Response`
  - `SyncChange`（payload 使用 `JsonElement`）
  - `SyncPullResponse`、`SyncEntityEnvelope`、`SyncTombstone`
- 扩展：`SyncStatusResponse` 增加 `serverCursor/capabilities/lastPushAck`

### 2) API 接口扩展
- 文件：`core/network/agent/AgentApi.kt`
- 新增：
  - `syncPush(...)`
  - `syncPull(since, limit)`

### 3) Client 封装扩展
- 文件：`core/network/agent/AgentClient.kt`
- 新增：
  - `syncPush(...)`
  - `syncPull(...)`

### 4) 同步管理器升级
- 文件：`core/data/DataSyncManager.kt`
- 新逻辑：
  - 收集本地增量后优先调用 `syncPush`
  - `syncPush` 成功后更新 `lastSyncTimestamp`
  - 若 `syncPush` 失败，自动回退到 `syncUpload`（v1）

## 验证结果

### 后端接口 smoke（通过）
- `POST /api/v1/sync/push`：返回 `accepted/applied/conflicts/server_cursor/results`
- `GET /api/v1/sync/pull`：返回 `changes/tombstones/next_cursor/server_time`
- `GET /api/v1/sync/status`：返回 `capabilities`，包含 `sync.push.v2`、`sync.pull.v2`

### Android 编译验证（通过）
- 在本地非云盘副本 `/tmp/HEl-local` 执行：

- 结果：`BUILD SUCCESSFUL`
- 结论：先前编译卡住主要由 OneDrive 路径读取异常导致，而非本次代码改动。
